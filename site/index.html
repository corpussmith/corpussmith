The provenance-first scholarly research workspace

Paste a title or a question. CorpusSmith expands it deterministically, harvests twenty academic sources, labels the trust of every record, and hands you a writing-ready export. No AI wrapper. No account. No lock-in.

Get Started GitHub
pip install -e .

What you type → what happens

$ corpussmith new my-thesis

Create a project

The wizard asks seven plain-English questions. Press Enter to accept any default. Your workspace is a folder — no hidden database, no server.

$ corpussmith search \
  --project my-thesis \
  "neural correlates of OCD"

Harvest sources

CorpusSmith classifies your input, expands it into a query plan, and harvests 20 academic APIs in parallel. Every record gets a trust label.

$ corpussmith export \
  --project my-thesis \
  --format bibtex

Export bibliography

BibTeX, CSL-JSON, or annotated Markdown — provenance labels ride with every record. Drop straight into Overleaf, Zotero, or Pandoc.

$ corpussmith build \
  --project my-thesis

Build knowledge dataset

Extract, chunk, and hash every document into a JSONL dataset. Feed it to RAG systems, train retrieval indices, or just search your reading pile.

Three promises

Deterministic

No model in the loop. Same input, same query plan, same record set — reproducible across machines and years.

Provenance-first

Every record carries a source type and trust label — peer-reviewed, preprint, grey literature, book, dataset. The label follows through all exports.

Scholar-first

Humanities-friendly. Multilingual. Unicode and accent-aware. Domain lexicons for six fields. Zero mandatory dependencies.
